⚠
Init system: OpenRC only. systemd is never used or referenced. All service management uses rc-update and rc-service.

STEP 01Preparation

1.1 Download the Minimal Install ISO

Browse the official autobuilds directory for the current AMD64 minimal ISO:

https://distfiles.gentoo.org/releases/amd64/autobuilds/current-install-amd64-minimal/

Download the file named install-amd64-minimal-<date>.iso. A .iso.asc GPG signature and .iso.sha256 checksum are also available alongside it for verification.

1.2 Write the ISO to a USB Drive

Replace /dev/sdX with your USB device — confirm with lsblk first and never write to your target NVMe.

dd if=install-amd64-minimal-<date>.iso of=/dev/sdX bs=4M status=progress oflag=sync

1.3 Boot into the Live Environment

Insert the USB drive, reboot, and enter your firmware boot menu (usually F12, F2, Del, or Esc). Select the Gentoo USB drive and press Enter at the boot prompt.

1.4 Verify UEFI Mode

EFI Stub boot requires UEFI. Confirm the environment booted in UEFI mode:

ls /sys/firmware/efi

If the directory exists and contains files such as efivars, you are in UEFI mode. If not, re-check your firmware settings. Ensure Secure Boot is disabled — unsigned EFI stub images will not boot with Secure Boot active unless you enroll your own signing keys.

1.5 Set Up Networking

Wired (DHCP — most common)

The live environment starts DHCP automatically on most hardware. Verify connectivity:

ping -c 3 gentoo.org

If that fails, bring the interface up manually. Note that modern Gentoo live media uses predictable interface names such as enp1s0, eno0, or enp5s0 — use ip link to find the exact name on your system:

ip link
ip link set enp1s0 up   # replace enp1s0 with your interface name
dhcpcd enp1s0

Wireless

iwconfig                                    # list wireless interfaces
iwlist wlp9s0 scan | grep ESSID             # scan (replace wlp9s0 with your interface)
wpa_passphrase "YourSSID" "YourPass" > /tmp/wpa.conf
wpa_supplicant -B -i wlp9s0 -c /tmp/wpa.conf
dhcpcd wlp9s0

1.6 Sync System Clock

An accurate clock is required for package signature verification:

chronyd -q 'server pool.ntp.org iburst'   # preferred
# or, as a fallback:
ntpdate pool.ntp.org

date  # confirm the result

mkdir -p /mnt/gentoo/efi

STEP 03Stage 3 Tarball

3.1 Download the Stage 3 (OpenRC)

Browse the current autobuilds directory for the latest OpenRC stage3:

https://distfiles.gentoo.org/releases/amd64/autobuilds/current-stage3-amd64-openrc/
⚠
Download the OpenRC stage3. The filename contains openrc — e.g. stage3-amd64-openrc-<date>T<time>Z.tar.xz. Do not download the systemd variant.
cd /mnt/gentoo
wget https://distfiles.gentoo.org/releases/amd64/autobuilds/current-stage3-amd64-openrc/stage3-amd64-openrc-<date>T<time>Z.tar.xz

Or browse interactively with links:

links https://distfiles.gentoo.org/releases/amd64/autobuilds/current-stage3-amd64-openrc/

3.2 Verify the Download handbook

The handbook provides .sha256 checksum files alongside each tarball. Download and verify before extracting:

wget https://distfiles.gentoo.org/releases/amd64/autobuilds/current-stage3-amd64-openrc/stage3-amd64-openrc-<date>T<time>Z.tar.xz.sha256

# Verify — output should end with "OK"
sha256sum --check stage3-amd64-openrc-<date>T<time>Z.tar.xz.sha256
💡
For the strongest verification, also download the .asc GPG signature and verify it against the Gentoo release engineering key. See the handbook's stage page for details.

3.3 Extract the Stage 3

tar xpvf stage3-*.tar.xz --xattrs-include='*.*' --numeric-owner -C /mnt/gentoo
ℹ
Flag meanings: x extract · p preserve permissions · v verbose · f file · --xattrs-include='*.*' preserve extended attributes (required for capabilities) · --numeric-owner preserve UID/GID exactly · -C /mnt/gentoo extract directly to the mount point regardless of current directory

STEP 04Configure Portage (make.conf)

nano /mnt/gentoo/etc/portage/make.conf

Replace the contents with the following. Adjust VIDEO_CARDS for your GPU:

make.conf
# Compiler flags — -march=native optimises for this exact CPU
COMMON_FLAGS="-march=native -O2 -pipe"
CFLAGS="${COMMON_FLAGS}"
CXXFLAGS="${COMMON_FLAGS}"
FCFLAGS="${COMMON_FLAGS}"
FFLAGS="${COMMON_FLAGS}"

# Parallel build jobs.  -l$(nproc) caps load average to prevent lockups.
MAKEOPTS="-j$(nproc) -l$(nproc)"

# Accept all licenses
ACCEPT_LICENSE="*"

# CPU feature flags — set correctly after running cpuid2cpuflags (see §4.2)
# CPU_FLAGS_X86="aes avx avx2 f16c fma3 mmx mmxext pclmul popcnt rdrand sse sse2 sse3 sse4_1 sse4_2 ssse3"

# Video card drivers:
#   Intel iGPU:           intel
#   AMD/Radeon:           radeonsi (or amdgpu)
#   NVIDIA open-source:   nouveau
#   NVIDIA proprietary:   nvidia  (requires separate setup)
#   None / virtual:       -* 
VIDEO_CARDS="intel"

# Optional: binary package host (saves compilation time)
# FEATURES="getbinpkg"
# BINPKG_FORMAT="gpkg"

4.1 Select Mirrors

# Install mirrorselect if not already available
emerge --ask app-portage/mirrorselect

# Pick geographically close mirrors interactively
mirrorselect -i -o >> /mnt/gentoo/etc/portage/make.conf

4.2 Set CPU_FLAGS_X86 handbook

The handbook recommends explicitly setting CPU_FLAGS_X86 for hand-written assembly intrinsics. This is separate from -march=native. After entering the chroot (§5), run:

emerge --ask app-portage/cpuid2cpuflags
cpuid2cpuflags

The output will be something like CPU_FLAGS_X86: aes avx avx2 ... — paste that line into /etc/portage/make.conf.

4.3 Optional: Binary Package Host handbook

Since late 2023 Gentoo provides an official binary package host for amd64. Enabling it dramatically reduces compilation time for large packages like GCC, LLVM, and Qt. To use it, uncomment these lines in make.conf:

FEATURES="getbinpkg"
BINPKG_FORMAT="gpkg"
💡
Binary packages are pre-built with a baseline x86-64 tuning. Packages that cannot be served from binaries will still be compiled locally. You can always disable this later and compile everything from source.

STEP 05Chroot

5.1 Copy DNS Configuration

cp --dereference /etc/resolv.conf /mnt/gentoo/etc/

5.2 Mount Necessary Filesystems

mount --types proc /proc /mnt/gentoo/proc
mount --rbind /sys /mnt/gentoo/sys && mount --make-rslave /mnt/gentoo/sys
mount --rbind /dev /mnt/gentoo/dev && mount --make-rslave /mnt/gentoo/dev
mount --bind /run /mnt/gentoo/run && mount --make-slave /mnt/gentoo/run

5.3 Enter the Chroot

chroot /mnt/gentoo /bin/bash
source /etc/profile
export PS1="(chroot) ${PS1}"

Your prompt should now show (chroot).

5.4 Mount the EFI Partition Inside the Chroot

mount /dev/nvme0n1p1 /efi
ℹ
The EFI partition was mounted at /mnt/gentoo/efi from outside the chroot. This command re-registers it inside the chroot's mount namespace so installkernel can write to /efi correctly.

STEP 06Portage Sync & Profile

6.1 Sync the Portage Tree

# Fast initial snapshot (recommended for first sync)
emerge-webrsync

# Then pull the latest incremental updates
emerge --sync

If Portage shows news items, read them — they often contain migration instructions:

eselect news list
eselect news read

6.2 Install cpuid2cpuflags handbook

Now is a good time to install and run this tool so you can set CPU_FLAGS_X86 before building anything:

emerge --ask app-portage/cpuid2cpuflags
cpuid2cpuflags

Add the output to /etc/portage/make.conf, for example:

echo 'CPU_FLAGS_X86="aes avx avx2 f16c fma3 mmx mmxext pclmul popcnt rdrand sse sse2 sse3 sse4_1 sse4_2 ssse3"' >> /etc/portage/make.conf

6.3 Select a Profile

eselect profile list
⚠
Select a profile that does not contain systemd in the name. The plain default/linux/amd64/23.0 profile uses OpenRC. Match the profile version to the stage3 you downloaded.
eselect profile set 1   # adjust the number to match your list
eselect profile show    # verify

6.4 Update the @world Set

Rebuild any packages whose USE flags changed with the new profile. This may take a while.

emerge --ask --verbose --update --deep --newuse @world

STEP 07Timezone, Locale & Hostname

7.1 Set the Timezone

echo "America/New_York" > /etc/timezone   # replace with your zone
emerge --config sys-libs/timezone-data

Full timezone list: /usr/share/zoneinfo/ — e.g. Europe/London, Asia/Tokyo, Australia/Sydney.

7.2 Configure Locales

nano /etc/locale.gen

Uncomment the locales you need, for example:

en_US ISO-8859-1
en_US.UTF-8 UTF-8
locale-gen
eselect locale list
eselect locale set 5   # adjust to the en_US.utf8 entry in your list

Set the locale explicitly for extra robustness:

cat > /etc/env.d/02locale << 'EOF'
LANG="en_US.UTF-8"
LC_COLLATE="C.UTF-8"
EOF

Reload the environment:

env-update && source /etc/profile && export PS1="(chroot) ${PS1}"

7.3 Set the Hostname

nano /etc/conf.d/hostname
hostname="gentoo"   # change to your preferred name

7.4 Configure /etc/hosts handbook

The handbook requires /etc/hosts to be set so the hostname resolves locally. This prevents startup warnings from tools that try to resolve the hostname:

nano /etc/hosts
/etc/hosts
127.0.0.1   localhost
::1         localhost
127.0.1.1   gentoo.localdomain gentoo   # replace "gentoo" with your hostname

STEP 08Kernel Installation (Distribution Kernel · EFI Stub · Fallback Boot)

This section installs the Gentoo distribution kernel (sys-kernel/gentoo-kernel) and boots it directly via UEFI using the UEFI fallback boot path. The firmware automatically boots any EFI image placed at /EFI/BOOT/BOOTX64.EFI on the EFI partition — no bootloader, no efibootmgr, no boot entries to manage.

This approach is chosen for simplicity of kernel updates: after each emerge, a single cp is all that is needed.

8.1 Get the Root Partition UUID

The root UUID must be compiled into the kernel command line. Get it now, before building anything:

blkid /dev/nvme0n1p3

Example output:

/dev/nvme0n1p3: UUID="a1b2c3d4-e5f6-7890-abcd-ef1234567890" BLOCK_SIZE="4096" TYPE="ext4" PARTUUID="..."

Copy the UUID= value — not PARTUUID.

8.2 Embed the Kernel Command Line

Because there is no bootloader or efibootmgr entry to supply parameters, the command line is compiled into the kernel image via CONFIG_CMDLINE. The distribution kernel's build system picks up snippet files from /etc/kernel/config.d/ automatically:

mkdir -p /etc/kernel/config.d

# Replace YOUR-ROOT-UUID-HERE with the UUID from blkid above
cat > /etc/kernel/config.d/cmdline.config << 'EOF'
CONFIG_CMDLINE="root=UUID=YOUR-ROOT-UUID-HERE rw"
CONFIG_CMDLINE_BOOL=y
EOF
💡
To add extra kernel parameters (e.g. quiet boot), append them inside the quotes after rw:
CONFIG_CMDLINE="root=UUID=... rw quiet loglevel=3"

8.3 Configure installkernel USE Flags

mkdir -p /etc/portage/package.use
echo "sys-kernel/installkernel efistub" > /etc/portage/package.use/installkernel

8.4 Emerge installkernel

emerge --ask sys-kernel/installkernel

8.5 Install Firmware

emerge --ask sys-kernel/linux-firmware
ℹ
linux-firmware supplies binary blobs for Wi-Fi adapters, GPUs, NVMe controllers, and other hardware that requires firmware at runtime. It is large but safe to install broadly.

8.6 Install the Distribution Kernel

emerge --ask sys-kernel/gentoo-kernel

This step:

  1. Downloads the kernel sources.
  2. Configures the kernel using Gentoo's maintained distribution config, merged with your snippets from /etc/kernel/config.d/ (including the embedded command line from §8.2).
  3. Compiles the kernel. This takes several minutes — longer on slower hardware.
  4. Calls installkernel which, with the efistub USE flag, places the compiled image on the EFI partition (typically /efi/EFI/Gentoo/bzImage.efi).

8.7 Locate the Kernel on the EFI Partition

find /efi -name "*.efi" -o -name "vmlinuz*" -o -name "bzImage*"

Expected output: /efi/EFI/Gentoo/bzImage.efi

8.8 Install to the UEFI Fallback Path

Copy the kernel to the standard UEFI fallback location. All compliant UEFI firmware boots this path automatically with no boot entry required:

mkdir -p /efi/EFI/BOOT

# Adjust source path if 'find' showed a different location above
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

ls -lh /efi/EFI/BOOT/BOOTX64.EFI   # confirm file is in place

8.9 Verify the Setup

# Confirm the fallback image exists
ls /efi/EFI/BOOT/BOOTX64.EFI

# Confirm the UUID is embedded in the kernel image
strings /efi/EFI/BOOT/BOOTX64.EFI | grep "root=UUID"

The second command should print your root=UUID=... command line. If it shows nothing, recheck /etc/kernel/config.d/cmdline.config and re-emerge the kernel.

💡
No efibootmgr is needed. The firmware finds BOOTX64.EFI on its own. There are no boot entries to manage or recreate after kernel updates.

STEP 09fstab Configuration (UUID-based)

Using UUIDs in /etc/fstab is critical for systems with multiple drives — device names like /dev/nvme0n1p3 can shift between boots if enumeration order changes. UUIDs never change.

9.1 Get UUIDs for All Three Partitions

blkid

Example output (your UUIDs will differ):

/dev/nvme0n1p1: UUID="AAAA-BBBB" TYPE="vfat" ...
/dev/nvme0n1p2: UUID="cccccccc-dddd-eeee-ffff-000000000000" TYPE="swap" ...
/dev/nvme0n1p3: UUID="a1b2c3d4-e5f6-7890-abcd-ef1234567890" TYPE="ext4" ...

9.2 Write /etc/fstab

nano /etc/fstab
/etc/fstab
# <filesystem>                              <mountpoint>  <type>  <options>           <dump> <pass>

# Root filesystem
UUID=a1b2c3d4-e5f6-7890-abcd-ef1234567890  /             ext4    defaults,noatime    0      1

# EFI System Partition
UUID=AAAA-BBBB                             /efi          vfat    defaults,noatime    0      2

# Swap
UUID=cccccccc-dddd-eeee-ffff-000000000000  none          swap    sw                  0      0
ℹ
Field notes:
noatime — skips access-timestamp updates on reads; improves SSD/NVMe performance and longevity.
dump column: 0 = skip dump backup, 1 = include.
pass column: 1 = root (checked first by fsck), 2 = others (checked after root), 0 = skip fsck.

STEP 10Networking & OpenRC Services

10.1 Configure a Network Interface

Find your interface name first:

ip link

Wired with DHCP (netifrc)

cd /etc/init.d
ln -s net.lo net.enp1s0              # replace enp1s0 with your interface name

cat >> /etc/conf.d/net << 'EOF'
config_enp1s0="dhcp"
EOF

rc-update add net.enp1s0 default

Wired with Static IP

cat >> /etc/conf.d/net << 'EOF'
config_enp1s0="192.168.1.100/24"
routes_enp1s0="default via 192.168.1.1"
dns_servers_enp1s0="1.1.1.1 8.8.8.8"
EOF

Alternative: dhcpcd (simpler)

emerge --ask net-misc/dhcpcd
rc-update add dhcpcd default

10.2 Install Essential System Tools

# System logger
emerge --ask app-admin/sysklogd
rc-update add sysklogd default

# Cron daemon
emerge --ask sys-process/cronie
rc-update add cronie default

# ext4 filesystem tools
emerge --ask sys-fs/e2fsprogs

# NTP time sync
emerge --ask net-misc/chrony
rc-update add chronyd default

10.3 Configure SSH (Optional)

rc-update add sshd default

useradd -m -G users,wheel,audio,video,usb,portage -s /bin/bash yourusername

# Confirm fallback kernel image is in place
ls -lh /efi/EFI/BOOT/BOOTX64.EFI

STEP 12Reboot

12.1 Exit the Chroot and Unmount

exit   # exit chroot

# Unmount in reverse order
umount -l /mnt/gentoo/dev
umount -l /mnt/gentoo/sys
umount /mnt/gentoo/proc
umount /mnt/gentoo/run
umount /mnt/gentoo/efi
umount /mnt/gentoo

12.2 Reboot

reboot

Remove the USB drive when the machine restarts. The UEFI firmware will boot Gentoo automatically via /EFI/BOOT/BOOTX64.EFI on the EFI partition.

12.3 First Boot Checklist

  • Log in as root (or your regular user if SSH is configured)
  • Verify networking: ping -c 3 gentoo.org
  • Check running services: rc-status
  • Confirm the correct kernel: uname -r
  • Update the system: emerge --sync && emerge --ask --update --deep --newuse @world
  • Check for Portage news: eselect news list && eselect news read

APPENDIX ATroubleshooting

System Does Not Boot

Boot back into the Gentoo live USB, re-mount, and chroot back in:

swapon /dev/nvme0n1p2
mount /dev/nvme0n1p3 /mnt/gentoo
mount /dev/nvme0n1p1 /mnt/gentoo/efi
mount --types proc /proc /mnt/gentoo/proc
mount --rbind /sys /mnt/gentoo/sys && mount --make-rslave /mnt/gentoo/sys
mount --rbind /dev /mnt/gentoo/dev && mount --make-rslave /mnt/gentoo/dev
mount --bind /run /mnt/gentoo/run && mount --make-slave /mnt/gentoo/run
cp --dereference /etc/resolv.conf /mnt/gentoo/etc/
chroot /mnt/gentoo /bin/bash
source /etc/profile && export PS1="(chroot) ${PS1}"
mount /dev/nvme0n1p1 /efi

Then diagnose:

# 1. Is the fallback image present?
ls -lh /efi/EFI/BOOT/BOOTX64.EFI

# 2. Is the UUID embedded correctly?
strings /efi/EFI/BOOT/BOOTX64.EFI | grep "root=UUID"

# 3. If missing, re-copy:
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

# 4. If UUID is wrong, fix the config and re-emerge:
nano /etc/kernel/config.d/cmdline.config
emerge sys-kernel/gentoo-kernel
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

Kernel Panic: Unable to Mount Root Filesystem

The root UUID compiled into the kernel is wrong. Boot from USB, chroot in, run blkid /dev/nvme0n1p3, correct /etc/kernel/config.d/cmdline.config, then re-emerge and re-copy:

emerge sys-kernel/gentoo-kernel
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

No Network After First Boot

ip link                   # find your actual interface name
ls /etc/init.d/net.*      # check which symlinks exist

# If the symlink for your interface is missing:
cd /etc/init.d
ln -s net.lo net.<your-interface>
rc-update add net.<your-interface> default
rc-service net.<your-interface> start

APPENDIX CUpdating the Kernel

With the UEFI fallback boot path approach, kernel updates require only a single cp after the emerge. No boot entries, no filename tracking.

# Sync and upgrade the full system
emerge --sync
emerge --ask --update --deep --newuse @world

# Or upgrade just the kernel
emerge --ask sys-kernel/gentoo-kernel

# Copy the new kernel to the fallback path — this is the only required step
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

# Verify
ls -lh /efi/EFI/BOOT/BOOTX64.EFI
strings /efi/EFI/BOOT/BOOTX64.EFI | grep "root=UUID"

# Clean up old versions
emerge --ask --depclean
ℹ
The cp is the only manual step after each update. The embedded UUID and command line in /etc/kernel/config.d/cmdline.config remain correct across kernel versions unless you repartition your drive.
