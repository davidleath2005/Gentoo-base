Gentoo AMD64 Installation Guide

OpenRC · EFI Stub · Distribution Kernel · NVMe · dwm / suckless

Every command is copy-paste ready unless a placeholder is shown. Read each section fully before running commands.

AMD64 (x86_64) OpenRC · no systemd EFI Stub · no bootloader Distribution Kernel AMD GPU · amdgpu/radeonsi dwm · dmenu · st
⚠
OpenRC only. systemd is never used or referenced. All service management uses rc-update and rc-service. Never select a systemd profile or stage3.
🔧
Revised guide. This version fixes the original kernel install failure. The key change: installkernel must be emerged with USE=efistub before emerging the kernel, and the EFI partition must be mounted at /efi inside the chroot before the kernel emerge runs. Both are now explicit required steps.

STEP 01Preparation

1.1 Download the Minimal Install ISO

Browse the official autobuilds for the current AMD64 minimal ISO:

https://distfiles.gentoo.org/releases/amd64/autobuilds/current-install-amd64-minimal/

Download install-amd64-minimal-<date>.iso. A .iso.asc GPG signature and .iso.sha256 checksum are available alongside it.

1.2 Write the ISO to USB

Confirm your USB device with lsblk first. Never write to your target NVMe.

dd if=install-amd64-minimal-<date>.iso of=/dev/sdX bs=4M status=progress oflag=sync

1.3 Boot the Live Environment

Insert the USB, reboot, enter your firmware boot menu (F12, F2, Del, or Esc). Select the Gentoo USB and press Enter at the boot prompt.

1.4 Verify UEFI Mode

ls /sys/firmware/efi

If the directory exists and contains efivars, you are in UEFI mode. If not, check your firmware settings. Disable Secure Boot — unsigned EFI stub images will not boot with it active.

1.5 Set Up Networking

Wired (DHCP)

ping -c 3 gentoo.org   # usually works automatically

If that fails, find your interface name and bring it up:

ip link
ip link set enp1s0 up   # replace enp1s0 with your interface
dhcpcd enp1s0

Wireless

iwlist wlp9s0 scan | grep ESSID
wpa_passphrase "YourSSID" "YourPass" > /tmp/wpa.conf
wpa_supplicant -B -i wlp9s0 -c /tmp/wpa.conf
dhcpcd wlp9s0

1.6 Sync System Clock

chronyd -q 'server pool.ntp.org iburst'
date   # confirm

STEP 02Disk Partitioning (NVMe)

⚠
Confirm your target NVMe device with lsblk before proceeding. These commands use /dev/nvme0n1 — adjust if your device name differs (e.g. /dev/nvme1n1).

2.1 Partition Layout

ℹ

2.3 Format

mkfs.fat -F 32 /dev/nvme0n1p1   # EFI must be FAT32
mkswap /dev/nvme0n1p2

2.4 Mount

mount /dev/nvme0n1p3 /mnt/gentoo
mkdir -p /mnt/gentoo/efi
mount /dev/nvme0n1p1 /mnt/gentoo/efi

STEP 04Configure Portage (make.conf)

This make.conf is tuned for your exact hardware: modern AMD CPU with AVX-512, AMD GPU (amdgpu/radeonsi), suckless desktop (dwm/dmenu/st), no systemd, no DE, latest software via ~amd64.

nano /mnt/gentoo/etc/portage/make.conf

Replace the entire contents with the following:

/etc/portage/make.conf
# /etc/portage/make.conf
# Target: AMD CPU w/ AVX-512 · AMD GPU · OpenRC · dwm/suckless · ~amd64

# ── COMPILER FLAGS ────────────────────────────────────────────────────
# -march=native auto-enables all supported CPU instructions (AVX-512, etc.)
# -O2 is stable. -O3 can be tried later but may cause subtle breakage.
# -pipe uses pipes instead of temp files — speeds up compilation.
COMMON_FLAGS="-O2 -pipe -march=native"
CFLAGS="${COMMON_FLAGS}"
CXXFLAGS="${COMMON_FLAGS}"
FCFLAGS="${COMMON_FLAGS}"
FFLAGS="${COMMON_FLAGS}"

# ── CPU FLAGS ─────────────────────────────────────────────────────────
# These enable hand-written SIMD intrinsics in packages that use them.
# Generated by: emerge app-portage/cpuid2cpuflags && cpuid2cpuflags
CPU_FLAGS_X86="aes avx avx2 avx512_bf16 avx512_bitalg avx512_vbmi2 avx512_vnni avx512_vp2intersect avx512_vpopcntdq avx512bw avx512cd avx512dq avx512f avx512ifma avx512vbmi avx512vl avx_vnni bmi1 bmi2 f16c fma3 mmx mmxext pclmul popcnt rdrand sha sse sse2 sse3 sse4_1 sse4_2 sse4a ssse3 vpclmulqdq"

# ── PARALLELISM ───────────────────────────────────────────────────────
# -j = number of parallel compile jobs (set to your logical thread count)
# -l = load average ceiling, prevents system lockup during big builds
# --jobs=4 in EMERGE_DEFAULT_OPTS allows parallel package emerges
MAKEOPTS="-j$(nproc) -l$(nproc)"
EMERGE_DEFAULT_OPTS="--ask --verbose --jobs=4 --load-average=$(nproc)"

# ── BRANCH ────────────────────────────────────────────────────────────
# ~amd64 = testing branch = latest software. Occasional breakage possible.
# Switch to amd64 for fully stable if preferred.
ACCEPT_KEYWORDS="~amd64"

# ── LICENSES ──────────────────────────────────────────────────────────
ACCEPT_LICENSE="-* @FREE @BINARY-REDISTRIBUTABLE"

# ── FEATURES ──────────────────────────────────────────────────────────
# parallel-fetch: download next package while building current
# candy: progress bar
# buildpkg: build binary packages (useful for reinstalls)
FEATURES="parallel-fetch candy buildpkg"

# ── AMD GPU ───────────────────────────────────────────────────────────
# amdgpu = modern AMD DRM driver (GCN3+)
# radeonsi = Mesa Gallium3D driver for AMD (required for OpenGL/Vulkan)
VIDEO_CARDS="amdgpu radeonsi"

# ── INPUT ─────────────────────────────────────────────────────────────
INPUT_DEVICES="libinput"

# ── GLOBAL USE FLAGS ──────────────────────────────────────────────────
# Philosophy: enable what dwm/X11/AMD GPU needs. Disable all DEs,
# systemd, bluetooth, and anything else not required for suckless setup.
USE="
    X
    alsa
    xinerama
    opengl
    vulkan
    opencl
    vaapi
    vdpau

    -systemd
    -elogind
    -wayland
    -gnome
    -kde
    -plasma
    -gtk4
    -bluetooth
    -pulseaudio
    -pipewire
    -cups
    -avahi
    -networkmanager
    -nls
    -ipv6
    -doc
    -debug
    -test
    -handbook

    unicode
    threads
    ssl
    pam
    udev
    dbus
    fontconfig
    truetype

# USE flag notes:
# - alsa: minimal audio. Add pulseaudio/pipewire later if needed.
# - dbus: required by many X11 apps even without a DE.
# - udev: device management via eudev (not systemd).
# - -nls: disables i18n. Remove if you need non-English locale support.
# - -ipv6: disables IPv6. Remove if your network uses it.
# - vaapi/vdpau: AMD GPU video decode acceleration.

# ── EFI STUB ──────────────────────────────────────────────────────────
# For installkernel to place the kernel on the EFI partition.
GRUB_PLATFORMS="efi-64"

# ── LOCALE ────────────────────────────────────────────────────────────
L10N="en"
LINGUAS="en"

4.1 Select Mirrors

# After entering the chroot, set mirrors via mirrorselect
emerge --ask app-portage/mirrorselect
mirrorselect -i -o >> /etc/portage/make.conf

STEP 05Chroot

5.1 Copy DNS

cp --dereference /etc/resolv.conf /mnt/gentoo/etc/

5.2 Mount Virtual Filesystems

mount --types proc /proc /mnt/gentoo/proc
mount --rbind /sys /mnt/gentoo/sys && mount --make-rslave /mnt/gentoo/sys
mount --rbind /dev /mnt/gentoo/dev && mount --make-rslave /mnt/gentoo/dev
mount --bind /run /mnt/gentoo/run && mount --make-slave /mnt/gentoo/run

5.3 Enter the Chroot

chroot /mnt/gentoo /bin/bash
source /etc/profile
export PS1="(chroot) ${PS1}"

Your prompt should now show (chroot).

5.4 Mount EFI Partition Inside the Chroot CRITICAL

mount /dev/nvme0n1p1 /efi
🔧
This step is required and must not be skipped. The EFI partition was mounted at /mnt/gentoo/efi from outside the chroot. This command makes it visible inside the chroot's mount namespace. Without it, installkernel cannot write the kernel image to /efi and the kernel postinst will fail with "Kernel install failed".
ℹ
Why does lsblk show two /efi entries? It's showing the same physical partition mounted in two namespaces — once from the live environment (/mnt/gentoo/efi), once from inside the chroot (/efi). This is normal and expected. It is not a problem.

STEP 08Kernel Installation Revised

This installs sys-kernel/gentoo-kernel and boots it directly via UEFI using the fallback boot path. The firmware automatically boots any EFI image at /EFI/BOOT/BOOTX64.EFI — no bootloader, no efibootmgr, no boot entries to manage.

🔧
Order matters. The steps below must be done in sequence: (1) set USE flag, (2) emerge installkernel, (3) confirm /efi is mounted, (4) emerge the kernel. Doing these out of order is what caused the original install failure.

8.1 Get the Root Partition UUID

The root UUID must be compiled into the kernel. Get it now:

blkid /dev/nvme0n1p3

Example output:

/dev/nvme0n1p3: UUID="a1b2c3d4-e5f6-7890-abcd-ef1234567890" TYPE="ext4" ...

Copy the UUID= value. Not PARTUUID.

8.2 Embed the Kernel Command Line

mkdir -p /etc/kernel/config.d

# Replace YOUR-ROOT-UUID-HERE with the exact UUID from blkid above
cat > /etc/kernel/config.d/cmdline.config << 'EOF'
CONFIG_CMDLINE="root=UUID=YOUR-ROOT-UUID-HERE rw"
CONFIG_CMDLINE_BOOL=y
EOF
💡
Double-check the UUID is correct before proceeding. A wrong UUID compiles into the kernel image and causes a kernel panic on boot. Verify: cat /etc/kernel/config.d/cmdline.config

8.3 Set installkernel USE Flag Critical — do first

mkdir -p /etc/portage/package.use
echo "sys-kernel/installkernel efistub" > /etc/portage/package.use/installkernel

8.4 Emerge installkernel Before the kernel

emerge --ask sys-kernel/installkernel
⚠
installkernel must be emerged before gentoo-kernel. If you emerge the kernel first, installkernel may not have the efistub USE flag applied and the postinst will fail. If you already emerged the kernel before this step, see §8.8.

8.5 Confirm /efi is Mounted Critical — check now

mount | grep efi    # must show /dev/nvme0n1p1 on /efi
ls /efi             # should show EFI/ directory (or be empty if first install)
✕
If /efi is not mounted, the kernel postinst will fail with "Kernel install failed". Mount it now: mount /dev/nvme0n1p1 /efi

8.6 Install Firmware

emerge --ask sys-kernel/linux-firmware
ℹ
Provides firmware blobs for AMD GPUs, NVMe controllers, Wi-Fi adapters, etc. Large but safe to install broadly. AMD GPU firmware is included and required for amdgpu.

8.7 Install the Distribution Kernel

emerge --ask sys-kernel/gentoo-kernel

This will:

  1. Download kernel sources
  2. Apply Gentoo's config merged with your /etc/kernel/config.d/ snippets (including the embedded UUID command line)
  3. Compile the kernel (several minutes)
  4. Call installkernel which, with efistub, places the image at /efi/EFI/Gentoo/bzImage.efi

8.8 Recovery: If Postinst Failed Previously If re-installing

If the kernel is already compiled but postinst failed, you do not need to recompile. Fix the prerequisites then run config-only:

# 1. Fix the USE flag and re-emerge installkernel
echo "sys-kernel/installkernel efistub" > /etc/portage/package.use/installkernel
emerge --newuse sys-kernel/installkernel

# 2. Confirm /efi is mounted
mount | grep efi
mount /dev/nvme0n1p1 /efi   # if not mounted

# 3. Re-run postinst only (no recompile)
emerge --config '=sys-kernel/gentoo-kernel-VERSION:VERSION'

8.9 Locate the Kernel

find /efi -name "*.efi" -o -name "vmlinuz*" -o -name "bzImage*"

Expected: /efi/EFI/Gentoo/bzImage.efi

8.10 Install to UEFI Fallback Path

mkdir -p /efi/EFI/BOOT
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI
ls -lh /efi/EFI/BOOT/BOOTX64.EFI

8.11 Verify

# Confirm fallback image exists
ls -lh /efi/EFI/BOOT/BOOTX64.EFI

# Confirm root UUID is baked into the image
strings /efi/EFI/BOOT/BOOTX64.EFI | grep "root=UUID"

The second command must print your root=UUID=... line. If it prints nothing, the cmdline config was not applied — fix /etc/kernel/config.d/cmdline.config and re-emerge the kernel.

💡
No efibootmgr needed. UEFI firmware finds BOOTX64.EFI automatically. There are no boot entries to manage after kernel updates — just run the cp in §8.10.

STEP 09fstab Configuration

9.1 Get All UUIDs

blkid

Example output:

/dev/nvme0n1p1: UUID="AAAA-BBBB" TYPE="vfat" ...
/dev/nvme0n1p2: UUID="cccccccc-dddd-eeee-ffff-000000000000" TYPE="swap" ...
/dev/nvme0n1p3: UUID="a1b2c3d4-e5f6-7890-abcd-ef1234567890" TYPE="ext4" ...

9.2 Write fstab

nano /etc/fstab
/etc/fstab
# <filesystem>                              <mountpoint>  <type>  <options>           <dump> <pass>

# Root filesystem
UUID=a1b2c3d4-e5f6-7890-abcd-ef1234567890  /             ext4    defaults,noatime    0      1

# EFI System Partition — mounted at /efi (not /boot)
UUID=AAAA-BBBB                             /efi          vfat    defaults,noatime    0      2

# Swap
UUID=cccccccc-dddd-eeee-ffff-000000000000  none          swap    sw                  0      0
ℹ
noatime skips access-time writes on reads — improves NVMe longevity.
pass 1 = root checked first by fsck · pass 2 = checked after root · pass 0 = skip fsck.

STEP 12Reboot

12.1 Exit Chroot and Unmount

exit   # leave chroot

# Unmount in reverse order
umount -l /mnt/gentoo/dev
umount -l /mnt/gentoo/sys
umount /mnt/gentoo/proc
umount /mnt/gentoo/run
umount /mnt/gentoo/efi
umount /mnt/gentoo

12.2 Reboot

reboot

Remove the USB when the machine restarts. UEFI firmware will boot Gentoo via /EFI/BOOT/BOOTX64.EFI automatically.

12.3 First Boot Checklist

  • Log in as root
  • Verify networking: ping -c 3 gentoo.org
  • Check services: rc-status
  • Confirm kernel version: uname -r
  • Confirm GPU is detected: lspci | grep -i vga
  • Update world: emerge --sync && emerge --ask --update --deep --newuse @world
  • Read news: eselect news list && eselect news read

APPENDIX ATroubleshooting

Kernel Postinst Failed: "Kernel install failed"

Almost always caused by either a missing efistub USE flag on installkernel, or /efi not being mounted. Fix and retry without recompiling:

echo "sys-kernel/installkernel efistub" > /etc/portage/package.use/installkernel
emerge --newuse sys-kernel/installkernel
mount /dev/nvme0n1p1 /efi
emerge --config '=sys-kernel/gentoo-kernel-VERSION:VERSION'
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

System Does Not Boot — Recovery Chroot

swapon /dev/nvme0n1p2
mount /dev/nvme0n1p3 /mnt/gentoo
mount /dev/nvme0n1p1 /mnt/gentoo/efi
mount --types proc /proc /mnt/gentoo/proc
mount --rbind /sys /mnt/gentoo/sys && mount --make-rslave /mnt/gentoo/sys
mount --rbind /dev /mnt/gentoo/dev && mount --make-rslave /mnt/gentoo/dev
mount --bind /run /mnt/gentoo/run && mount --make-slave /mnt/gentoo/run
cp --dereference /etc/resolv.conf /mnt/gentoo/etc/
chroot /mnt/gentoo /bin/bash
source /etc/profile && export PS1="(chroot) ${PS1}"
mount /dev/nvme0n1p1 /efi

Diagnose:

# Is fallback image present?
ls -lh /efi/EFI/BOOT/BOOTX64.EFI

# Is root UUID embedded?
strings /efi/EFI/BOOT/BOOTX64.EFI | grep "root=UUID"

# If image is missing, copy it:
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

Kernel Panic: Unable to Mount Root

Wrong UUID compiled in. Fix and re-emerge:

blkid /dev/nvme0n1p3
nano /etc/kernel/config.d/cmdline.config   # fix the UUID
emerge sys-kernel/gentoo-kernel
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

No Network After First Boot

ip link
ls /etc/init.d/net.*

# If symlink for your interface is missing:
cd /etc/init.d
ln -s net.lo net.<your-interface>
rc-update add net.<your-interface> default
rc-service net.<your-interface> start

APPENDIX BOpenRC Quick Reference

ℹ
There is no systemctl. All service management uses rc-service and rc-update.
TaskCommand
Start a servicerc-service <name> start
Stop a servicerc-service <name> stop
Restart a servicerc-service <name> restart
Enable on bootrc-update add <name> default
Disable on bootrc-update del <name> default
List all servicesrc-status
List services in runlevelrc-update show

APPENDIX CUpdating the Kernel

With the UEFI fallback path approach, kernel updates need only a single cp after emerge. No boot entries, no filenames to track.

# Sync and upgrade world
emerge --sync
emerge --ask --update --deep --newuse @world

# Or just the kernel
emerge --ask sys-kernel/gentoo-kernel

# Deploy to fallback path — this is the only required step
cp /efi/EFI/Gentoo/bzImage.efi /efi/EFI/BOOT/BOOTX64.EFI

# Verify
strings /efi/EFI/BOOT/BOOTX64.EFI | grep "root=UUID"

# Clean up old kernel versions
emerge --ask --depclean
ℹ
The embedded UUID in /etc/kernel/config.d/cmdline.config remains correct across kernel versions unless you repartition. The cp is the only manual step after each update.

APPENDIX DPost-install: dwm / X11 / AMD GPU New

After a successful first boot, install the X server and suckless tools. The make.conf from §4 already sets the correct USE flags and AMD GPU drivers.

D.1 Install Xorg

emerge --ask x11-base/xorg-server x11-apps/xinit x11-apps/xrandr

D.2 Install AMD GPU Userspace (Mesa)

Mesa provides OpenGL, Vulkan, and VAAPI for your AMD GPU. It should pull in automatically with Xorg given your VIDEO_CARDS setting, but emerge it explicitly to be sure:

emerge --ask media-libs/mesa x11-drivers/xf86-video-amdgpu
💡
Verify the GPU is picked up after X is running: glxinfo | grep "OpenGL renderer" — should show your AMD GPU, not llvmpipe (software fallback).

D.3 Install Build Dependencies for suckless

emerge --ask \
  x11-libs/libX11 \
  x11-libs/libXft \
  x11-libs/libXinerama \
  media-libs/fontconfig \
  media-fonts/noto \
  dev-util/pkgconf

D.4 Build dwm, dmenu, st

Clone, configure, and install the suckless tools from source:

cd /usr/local/src

# dwm
git clone https://git.suckless.org/dwm
cd dwm
cp config.def.h config.h   # edit config.h to customise bindings, fonts, etc.
make && make install
cd ..

# dmenu
git clone https://git.suckless.org/dmenu
cd dmenu
cp config.def.h config.h
make && make install

D.5 Create .xinitrc

cat > ~/.xinitrc << 'EOF'
#!/bin/sh
# Set keyboard repeat rate
xset r rate 300 50

# Start dwm
exec dwm
EOF
chmod +x ~/.xinitrc

D.6 Start X

startx
💡
Add [[ -z $DISPLAY && $XDG_VTNR -eq 1 ]] && startx to your ~/.bash_profile to start X automatically on TTY1 login.
